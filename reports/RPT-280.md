## Problems Found

### P-1 — ARCHITECTURE.md V6.3-Section dokumentiert `ON CONFLICT (slug, version)`, echter Constraint ist nur `slug`

**Severity:** Medium (Doku-Reality-Gap, in MT-1 konkret bemerkt und sauber umgeschifft).

**Befund:** ARCHITECTURE.md V6.3-Section Zeile 5975 zitiert `ON CONFLICT (slug, version) DO UPDATE SET ...`. Das `template`-Schema (MIG-021 Zeile 12) hat aber nur `slug text NOT NULL UNIQUE` — kein zusammengesetzter UNIQUE-Constraint `(slug, version)`. Existierende Seed-Migrations (MIG-029 exit_readiness, MIG-046 demo) nutzen `ON CONFLICT (slug) DO NOTHING`.

**Behebung:** Migration 093 nutzt `ON CONFLICT (slug) DO UPDATE SET name, version, description, blocks, metadata, updated_at`. Damit ist die Migration idempotent (Re-Apply ist No-Op auf gleichem Inhalt) UND Update-faehig (bei spaeterer Workshop-Output-v2 ueberschreibt eine neue Migration den partner_diagnostic-Eintrag). Header-Kommentar in 093 dokumentiert die Abweichung explizit, MIGRATIONS.md MIG-037 ebenfalls.

**Folge:** Eine spaetere V6.4+ Migration `094_partner_diagnostic_v2.sql` mit `version='v2'` wuerde via `ON CONFLICT (slug) DO UPDATE` den v1-Eintrag UEBERSCHREIBEN (kein Parallel-Halten von v1+v2). Falls Parallel-Halten gewuenscht: vorher den UNIQUE-Constraint auf `(slug, version)` umstellen. Out-of-Scope V6.3.

### P-2 — Slice-File-Drift: Verweis auf `Migration 091a` als Template-Seed-Slot

**Severity:** Low (bekannt aus RPT-279 P-2).

**Befund:** SLC-105-Slice-File (Zeile 26) schlaegt `091a_v6_partner_diagnostic_template_seed.sql` vor. Slot `091a` ist aber bereits vergeben fuer Style-Guide-Brand-Color-Fix (`091a_align_strategaize_default_brand_color_styleguide_v2.sql`).
