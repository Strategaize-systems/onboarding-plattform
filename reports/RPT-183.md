### Medium

#### M-3 — ai_cost_ledger Cost-Tracking fehlte (Silent-Failure)

**Was:** `ai_cost_ledger.role` CHECK-Constraint enthielt vor Migration 088 nur 11 V1-V4-Roles. Worker schreibt `role: "walkthrough_pii_redactor"` — nicht erlaubt → INSERT scheitert mit constraint-violation. Der Worker pruefte den `{ error }` aus dem Supabase-INSERT-Result NICHT, also wurde der Fehler silent verschluckt. 4 erfolgreiche PII-Redaction-Live-Smokes erzeugten 0 Cost-Ledger-Eintraege.

**Wirkung:** Pipeline funktioniert weiter (PII-Redaction + Status-Wechsel + Folge-Job-Enqueue), aber kein Cost-Tracking fuer V5 Option 2. Compliance/Audit-Risiko mittelschwer (DSGVO sieht Cost-Audit nicht vor, aber interne Cost-Controlling-Dashboards waeren blind). Plus: das Silent-Failure-Pattern ist ein Code-Quality-Bug, der sich auf SLC-077 + SLC-078 ausgewirkt haette.

**Fix (in derselben /qa-Session):**
- **Migration 088** (`sql/migrations/088_v5opt2_ai_cost_ledger_roles.sql`, live appliziert auf Hetzner via base64-Pipe + `psql -U postgres`): CHECK-Erweiterung um drei V5-Option-2-Roles (`walkthrough_pii_redactor`, `walkthrough_step_extractor`, `walkthrough_subtopic_mapper`). Die letzten zwei sind Vorbereitungs-Anker fuer SLC-077 + SLC-078 — kein Drift bei spaeterem Worker-Deploy.
- **Code-Fix in `handle-redact-pii-job.ts`** (commit `a3f4aff`): `{ error: costError } = await adminClient.from("ai_cost_ledger").insert({...})`, bei Fehler `captureWarning` mit Job-/Session-Kontext. Cost-Tracking ist nice-to-have und KEIN Pipeline-Blocker — Worker faehrt im Erfolgspfad weiter, aber Failure ist sichtbar im Log.
- **Vitest erweitert** auf 9 Tests (1 neuer "captures warning + completes successfully when ai_cost_ledger INSERT fails"-Pfad).

**Verifikation post-Hotfix (offen bis User-Coolify-Redeploy auf commit `a3f4aff`):**
- Worker-Container neu starten.
- Eine vierte walkthrough_session auf `redacting` setzen + neuer ai_jobs walkthrough_redact_pii.
- Job laeuft durch → ai_cost_ledger zeigt 1 Eintrag mit role='walkthrough_pii_redactor', feature='walkthrough_pii_redaction'.

**Zukunfts-Empfehlung:** Skill-Improvement im Dev-System: bei Supabase-Client INSERTs immer `{ error }` destructuren + bei nicht-fatal-Pfaden `captureWarning` statt silent ignore. Das gilt fuer alle Worker-Code-Pfade. Aktuell zeigt `dialogue/handle-extraction-job.ts` auch das Silent-Failure-Pattern (Zeile 206-216, ai_cost_ledger ohne error-check). Sollte als generische IMP-Verbesserung im Onboarding-Repo kommen.
