# SLC-V9.1-E — Forward-Address Single-Mailbox-Display-Fix (ISSUE-098)

## 1. Outcome

Code + /qa Code-Side **PASS**. ISSUE-098 (Setup-UI zeigt bouncende
Catchall-Adresse) ist code-seitig behoben; die Live-UI zeigt die korrekte Adresse
erst nach Founder-ENV-Set + Redeploy → ISSUE-098 bleibt bis dahin `open`.

**Verdict:** 0 Blocker / 0 High / 0 Medium / 1 Low.

## 2. Root cause

Die Forward-Setup-UI baute das Weiterleitungs-Ziel an 3 Stellen hart als
`bulk-<slug>@bulk.strategaizetransition.com` (Catchall-Form). Diese Subdomain hat
keinen MX-Record → Mails bouncen. As-built (DEC-205/206) ist Single-Mailbox: alle
Forwards laufen ueber EIN reales IONOS-Postfach. Besonders kritisch: `sendTestEmail`
mailte ebenfalls an die bouncende Adresse → Test-Mail garantiert `received=false`.

## 3. Fix (DEC-211)

ENV-Modus-Schalter `INBOUND_MAILBOX_ADDRESS`, zentralisiert in einem Resolver:
- gesetzt → Single-Mailbox: Adresse = reales Postfach (slug-unabhaengig)
- nicht gesetzt → Catchall-Fallback (Zukunft, rueckwaertskompatibel)
