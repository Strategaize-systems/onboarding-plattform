## 7. Problems found
- **MT-5 caught a real compile bug:** `firstSentence` used the `/s` (dotAll) regex flag, which fails under this repo's tsconfig target (`TS1501`, needs es2018+). Fixed by removing the flag — cross-newline matching wasn't needed; the first-line fallback covers the multiline case. All 34 tests still GREEN after the fix. (Deviation Rule 1 / 3: auto-fixed, reported here.)
- **"Log-Hinweis" for ai_draft remap (spec MT-1) intentionally not implemented in the lib.** The spec mentions a log hint when `ai_draft`→`observation`. The emitter is a pure, isolated module with 0 worker imports (AC-A-6) and no logger; emitting a console side-effect would pollute the pure function and tests. The remap is deterministic and documented in code. **Deferred to SLC-V9.7-B**, where the worker (which owns the logger) can log when it encounters `ai_draft` rows in the source set. Noted so it is not lost.
- **Filename uniformity vs. ASCII sketch:** the architecture's ASCII bundle layout shows `diagnosis-<id8>.md` (no slug), but the spec's `conceptFilename(type, title, id)` is uniform `<type>-<slug>-<id8>.md`. I followed the explicit function contract (uniform) → diagnosis files are `diagnosis-diagnose-<block_key>-<id8>.md` (slightly redundant but deterministic and consistent). No functional impact; cross-links in SLC-V9.7-B consume `OkfConcept.path` directly.
