## Live-Bug-Fix-Log

### Bug 1 — capture_mode NULL-Filter (commit d6869d0)
**Symptom:** Page zeigte "Noch keine GF-Erhebung" trotz existierender Session 64ad04eb.
**Root cause:** Demo-Owner-Session hatte `capture_mode = NULL` (Legacy vor V4). `.neq("capture_mode", "employee_questionnaire")` filtert NULL-Rows aus (Postgres-NULL-Vergleich = UNKNOWN, nicht TRUE).
**Fix:** `.or("capture_mode.is.null,capture_mode.neq.employee_questionnaire")` inkludiert NULL.

### Bug 2 — Session-Priorisierung mit Bridge-Runs (commit 04e16f3)
**Symptom:** Demo-Admin hat 2 GF-Sessions (juengere leer, aeltere mit 2 bridge_runs). Page nahm immer die juengste (leer) → "Noch kein Bridge-Lauf".
**Root cause:** Naive `.order(started_at desc).limit(1)` ignorierte semantische Relevanz.
**Fix:** Page laedt alle eigenen GF-Sessions, prueft welche bereits bridge_runs hat, priorisiert diese. Fallback: juengste Session.

### Bug 3 — RLS-Policy fehlt fuer tenant_admin auf profiles (Migration 076, commit f617dc1)
**Symptom:** Mitarbeiter-Dropdown im Edit-Dialog leer, ProposalCards zeigten "Noch nicht zugeordnet" trotz gesetzter `proposed_employee_user_id`.
**Root cause:** public.profiles hatte nur `admin_full_profiles` (strategaize_admin) und `user_select_own_profile` (eigene Row). tenant_admin konnte Mitarbeiter-Profiles nicht lesen.
**Fix:** Neue Policy `tenant_admin_select_tenant_profiles` (FOR SELECT TO authenticated, USING role='tenant_admin' AND tenant_id=user_tenant_id).
**Hinweis:** Auch /admin/team Aktive-Mitarbeiter-Liste war pre-Fix silent-empty.

### Bug 4 — auth-Schema USAGE-Grant fehlt (Migration 077, commit 6b6abcf)
**Symptom:** Bug 3 gefixt, aber Effekt blieb aus — Mitarbeiter-Liste weiterhin leer.
**Root cause:** `authenticated` Postgres-Role hatte kein USAGE auf das `auth`-Schema. Cross-Schema-Function-Calls (`auth.user_role()`/`auth.user_tenant_id()`) in der Policy-Expression konnten nicht aufgeloest werden, Policy evaluierte stillschweigend FALSE. Standard-Supabase-Setup, war auf der Onboarding-DB nicht gesetzt.
**Fix:** `GRANT USAGE ON SCHEMA auth TO authenticated, anon`.
**Lesson:** Bei Self-hosted-Supabase-Onboarding-Health-Check sollte `nspacl` auf `auth` enthalten `authenticated=U/...` — sonst greifen viele RLS-Policies still nicht.

### Bug 5 — Type-Re-Export aus "use server"-File (commit 2b730d0)
**Symptom:** Klick "Speichern & Approven" → 500 mit `ReferenceError: EditedProposalPayload is not defined` aus `.next/server/chunks`.
**Root cause:** Next.js "use server"-Files duerfen nur async-function-Exports enthalten. `export type { EditedProposalPayload }` brach die Server-Action-Compilation zur Laufzeit.
**Fix:** Re-Export entfernt. EditDialog importiert Type direkt aus action-helpers.

### Bug 6 — Toaster nicht im Layout gemounted (commit 3b7c152)
**Symptom:** Leeres Reject im AlertDialog produzierte keinen Toast-Error, Dialog blieb stumm stehen. Erfolgsfaelle wurden nur durch revalidatePath sichtbar.
**Root cause:** Sonner-`<Toaster />` aus `@/components/ui/sonner` war in `src/app/layout.tsx` nicht gemounted. Alle `toast.error()`/`toast.success()`-Calls feuerten ins Leere.
**Fix:** `<Toaster />` ins Root-Layout. Behebt Toast-Probleme global (Bridge + Team-Invitations).

### Bug 7 — Hydration Mismatch durch TZ-Diskrepanz (commit 89836b4)
**Symptom:** Console zeigte React Error #418 (Hydration Mismatch).
**Root cause:** `BridgeRunList` formatierte `run.created_at` mit `toLocaleString("de-DE")` sowohl beim Server-Side-Render des Client-Components als auch im Browser. Server-TZ (UTC) vs Browser-TZ (Europe/Berlin) erzeugten unterschiedliche Strings ("07:16" vs "09:16") → Mismatch.
**Fix:** Server-Component formatiert mit `Intl.DateTimeFormat({ timeZone: "Europe/Berlin" })` und reicht `formattedCreatedAt`-String an Client. Gleicher String auf Server und Client.
