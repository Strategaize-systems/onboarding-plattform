# Requirements V9.5 — Bulk-Import Deep-Extraction

Baseline based on /discovery RPT-452 (scope cut locked with founder). Turns the discovery cut into an execution-ready V1 baseline.

## Scope (3 features)

- **FEAT-080 Cross-Thread-Synthese-Stage** (BL-159, core): additive stage between `email_bulk_run.status` `pattern_extracted` → `synthesized`/`curating`. Input = a run's raw `email_pattern` rows; output = consolidated, deduped, evidence-weighted candidate units. Existing flat per-thread extractor untouched.
- **FEAT-081 Bounded Critic/Quality-Gate** (BL-160): Analyst↔Challenger *principle* (not code) on the synthesized output, fixed 1–2 passes, hard cost-cap preserved, flags/discards low-quality units before GF curation.
- **FEAT-082 Bedrock Model-Default Cleanup** (BL-161, ISSUE-100): 4 files → eu-Sonnet-4 default (`email-pattern.ts:51`, `bedrock-haiku/index.ts:42`, `v8-1-augmentation/augment.ts:44-46`, `bulk-email/ai-assisted-setup.ts:24`). ENV-override mechanics unchanged.

## Integration contract (code-grounded)

- Status machine (MIG-106): `…pattern_extracting → pattern_extracted → curating → importing → completed|failed`. New stage slots after `pattern_extracted`, before `curating` (new values `synthesizing`/`synthesized`).
- Curation promotion (`curation/actions.ts` `importAcceptedPatterns`): accepted/edited `email_pattern` → `knowledge_unit` INSERT (Markdown body + metadata JSONB source-attribution) + `imported_knowledge_unit_id`. **Must stay unbroken** — synthesis only feeds higher-quality cards in.
- Cost-cap (`src/lib/bulk-email/cost-cap.ts` `checkLiveCapInWorker` / `V9_BULK_EMAIL_RUN_CAP_EUR`): the new synthesis/critic calls **must** fall under the same run hard-cap.
- `email_pattern` columns available as synthesis input: title, description, evidence_snippets (jsonb), themes (text[]), confidence, suggested_section.
