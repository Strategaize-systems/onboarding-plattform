## Wiring chain — verified end-to-end

```
[/dashboard page.tsx]
   -> getInactiveEmployeesCount(supabase, tenantId)
      -> employee_invitation (status='accepted')
      -> block_checkpoint (LEFT antijoin via Set)
   -> InactiveEmployeesCard(props)
      -> Tooltip (DEC-058 Text)
      -> Link href=/admin/team?filter=inactive

[/admin/team page.tsx]
   -> searchParams.filter -> parseFilter -> "all"|"inactive"
   -> employee_invitation + profiles + block_checkpoint
      -> enrichedEmployees mit lastBlockSubmit
   -> Filter-Tabs (Link href Toggle)
   -> Tabelle mit Status-Badge

[/dashboard/settings page.tsx]
   -> createAdminClient().from("user_settings").select(...).eq("user_id", auth.uid())
   -> RemindersOptOutToggle(initial=DB-Wert)
      -> useTransition + toggleRemindersOptOut(value)

[Server-Action toggleRemindersOptOut]
   -> createClient().auth.getUser()  // Auth-Check
   -> createAdminClient().from("user_settings")
        .update({ reminders_opt_out, updated_at })
        .eq("user_id", user.id)      // Service-Role + expliziter Filter (IMP-214)
   -> revalidatePath("/dashboard/settings")
   -> Toast (sonner)
   -> Reload zeigt persistierten Wert

[RLS-Schutzschicht (Defense-in-Depth)]
   -> user_settings_own_update Policy: USING (user_id = auth.uid())
   -> Test verifiziert: cross-user UPDATE => 0 rows affected
```

Alle Glieder gegen Live-Daten gepruft. Kein Mismatch zwischen Component-,
API-, DB- und State-Layer.
