# RPT-618 — ISSUE-119 Fix: /capture/new ZodError-Crash

## Scope
Kleiner Backend-Hotfix (kein Slice-Feature, keine Migration, kein DB-Change). Behebt ISSUE-119: `/capture/new` warf für JEDEN `tenant_admin` einen ZodError, weil `listTemplates()` alle `template`-Rows gegen `TemplateRowSchema` mappt und beim ersten ungültigen Row (`.parse`) wirft. Live-DB: 2 von 22 Rows (`partner_diagnostic`, `exit-readiness-teaser-v1`) haben Legacy-Blocks (title String/fehlt, keine IDs).

## Fix (TDD RED→GREEN)
- `src/lib/db/template-queries.ts` (`listTemplates`): `.map(row => TemplateRowSchema.parse(row))` → per-Row `safeParse`-Schleife. Valide Rows werden gesammelt; ungültige übersprungen + via `captureWarning("template-queries.listTemplates", { slug, issues })` geloggt (Reuse `src/lib/logger.ts`). DB-Fehler werfen weiterhin. `getTemplateBySlug`/`getTemplateById` unverändert (Einzel-Lookup valider Templates).
- Damit fallen die 2 Legacy-Rows automatisch aus dem Picker; Lösung ist zukunftssicher gegen weiteren Schema-Drift.
