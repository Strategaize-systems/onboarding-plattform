## Problems Found

### P-1 — Slice-File-Drift: `template.metadata` und `knowledge_unit.metadata` Spalten existieren NICHT (kritischer Schema-Befund)

**Severity:** Medium (zwischen Slice-Plan und Schema-Reality).

**Befund:** Die SLC-105-Slice-Datei (`/slices/SLC-105-...md` Zeile 16, Zeile 47-50, Zeile 110-112) referenziert `template.metadata.usage_kind` und `knowledge_unit.metadata={ score, comment, score_rule_version }` als ob diese Spalten existieren wuerden. Sie tun das NICHT — Migration 091 hat trotz dem irrefuehrenden Dateinamen `091_v6_partner_branding_AND_template_metadata.sql` KEINE `template.metadata` Spalte angelegt (Verifikation: Grep nach `ALTER TABLE template` in Migration 091 = 0 Treffer, Grep nach `metadata` in Migration 091 = 1 Treffer im Comment-Header).

Existierende `template`-Spalten:
- aus MIG-021: `id`, `slug`, `name`, `version`, `description`, `blocks`, `created_at`, `updated_at`
- aus MIG-045: `sop_prompt`, `owner_fields`
- aus MIG-051: `diagnosis_schema`, `diagnosis_prompt`

Existierende `knowledge_unit`-Spalten:
- aus MIG-021: `id`, `tenant_id`, `capture_session_id`, `block_checkpoint_id`, `block_key`, `unit_type`, `source`, `title`, `body`, `confidence`, `evidence_refs`, `status`, `created_at`, `updated_at`, `updated_by`
- KEINE `metadata`-Spalte irgendwo.

**Behebung:** Migration 093 (geplant in MT-1) fuegt beide Spalten als idempotenten `ALTER TABLE ADD COLUMN IF NOT EXISTS metadata jsonb NOT NULL DEFAULT '{}'::jsonb` hinzu. DEC-127 fixiert Migration-Nummer 093. ARCHITECTURE.md V6.3-Section dokumentiert beide Schema-Erweiterungen explizit. Damit ist die Slice-Reality auf Schema-Stand gebracht.

**Konsequenz fuer Slice-File:** Optional — die SLC-105-Datei koennte in MT-1 mit einem `Hinweis: Migration 093 fuegt template.metadata + knowledge_unit.metadata Spalten hinzu (DEC-127, RPT-279)` angereichert werden. Strenggenommen nicht noetig, weil MT-1 die Migration anlegt und damit den Stand klarstellt.
