### Findings

#### High (fixed during QA)

- **H-1: Zod schema mismatch — TemplateBlockSchema.title** in `/src/lib/db/template-queries.ts:7`
  Schema defined `title: z.string()` but seed data stores title as `{de: "...", en: "...", nl: "..."}`. Would cause runtime `ZodError` on any `getTemplateBySlug()` call. **Fixed:** Changed to `z.record(z.string(), z.string())`. Also added missing `weight` field. Commit `3d2a9f1`.

#### Low

- **L-1: Legacy Blueprint references remain in src/ (16 occurrences)**
  Files: `email.ts` (2x), `catalog-client.tsx` (4x), `api/admin/catalog/snapshots/route.ts` (5x), `api/admin/runs/[runId]/export/route.ts` (4x), `validations.ts` (1x). All are in legacy Blueprint catalog/export code that predates the new template system. `blueprint_version` is a DB column name — rename requires schema migration. Not SLC-003 scope. These will be addressed when the legacy catalog system is replaced or removed.

- **L-2: Tests cannot run locally**
  Template query tests exist (5 cases) but require `TEST_DATABASE_URL` pointing to a Postgres instance with the seeded template. No local SSH tunnel configured. Consistent with prior Low Finding L-2 from RPT-016.
