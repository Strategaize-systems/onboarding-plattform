# V6 SLC-104 /doctor MT-7 Live-Smoke — Middleware-Bug Public-Routes

## Outcome summary

**Status: Bugfix code-complete, Live-Verifikation deferred auf User-Coolify-Redeploy.**

Nach Coolify-Redeploy von HEAD `d5d8c2b` (RPT-229+230 MT-7-Code) wurde der optionale Live-Smoke aus dem Session-Handoff gefahren. Beide URLs gaben 307 Temporary Redirect → `/login` statt der spezifizierten 404/400. Root-Cause-Analyse fuehrte zur Proxy-Middleware. Fix als 1-Zeiler eingespielt, gepusht. User-Redeploy-Pflicht.

## Symptom

```
$ curl -sSI https://onboarding.strategaizetransition.com/api/partner-branding/00000000-0000-0000-0000-000000000001/logo
HTTP/1.1 307 Temporary Redirect
Location: /login

$ curl -sSI https://onboarding.strategaizetransition.com/api/partner-branding/not-a-uuid/logo
HTTP/1.1 307 Temporary Redirect
Location: /login
```

Erwartet (laut Slice-Spec Section F + RPT-229):
- `/api/partner-branding/<gueltige-uuid-ohne-branding>/logo` → `404 NOT_FOUND`
- `/api/partner-branding/not-a-uuid/logo` → `400 BAD_REQUEST`

`/api/health` antwortet weiterhin korrekt mit `200 OK` — Symptom ist auf `/api/partner-branding/*` beschraenkt.

## Root-Cause

`src/lib/supabase/middleware.ts:49` (Pre-Fix):

```typescript
const isApiHealth = pathname === "/api/health";
const isApiCron = pathname.startsWith("/api/cron/");
const isApiUnsubscribe = pathname.startsWith("/api/unsubscribe/");

if (!user && !isPublicPath && !isApiHealth && !isApiCron && !isApiUnsubscribe) {
  const url = request.nextUrl.clone();
  url.pathname = "/login";
  return NextResponse.redirect(url);
}
```

Die Proxy-Middleware (`src/proxy.ts` → `updateSession`) prueft `auth.getUser()` und redirected unauthenticated Requests auf `/login`, sofern der Pfad nicht in der Public-Liste steht. `/api/partner-branding/*` war nicht gelistet, obwohl der Route-Handler selbst kein Auth-Gate hat und die Slice-Spec Section F (Zeile 184-186) explizit fordert:

> "Auth ist optional: Logo wird auch fuer anonyme Login-Page geladen (Branding bevor Login)."

DEC-109-Tradeoff (RPC SECURITY DEFINER mit anon-EXECUTE) baut darauf auf, dass die Route public erreichbar ist.

## Warum Code-Side-QA (RPT-230) das nicht aufgedeckt hat

Die 11 Vitest in `src/app/api/partner-branding/[partner_tenant_id]/logo/__tests__/route.test.ts` rufen den GET-Handler direkt auf. Vitest umgeht Next.js Middleware komplett — `proxy.ts` wird nur im Edge-Runtime einer echten Request-Kette ausgefuehrt. Das Testing-Setup-Pattern (RPT-229 dokumentiert) hat keinen Live-Smoke gegen die deployed Edge-Middleware, weil die Coolify-DB-Test-Strategie (`reference_coolify_test_setup.md`) auf Server-side Vitest ausgerichtet ist.

**Skill-Improvement-Kandidat:** /qa.md sollte fuer NEUE API-Routes immer einen optionalen Live-Smoke gegen die Production-URL fordern, um Middleware-/Edge-Layer-Probleme aufzudecken. Wird im Handoff als IMP erfasst.

## Fix

`src/lib/supabase/middleware.ts` (Commit `b256ed9`):

```typescript
const isApiHealth = pathname === "/api/health";
const isApiCron = pathname.startsWith("/api/cron/");
const isApiUnsubscribe = pathname.startsWith("/api/unsubscribe/");
// SLC-104 MT-7: Partner-Branding-Logo wird auch fuer anonyme Login-Page geladen
// (Branding ab erstem Pixel, vor Login). Route hat eigenen UUID-Guard + RPC-Check.
const isApiPartnerBranding = pathname.startsWith("/api/partner-branding/");

if (!user && !isPublicPath && !isApiHealth && !isApiCron && !isApiUnsubscribe && !isApiPartnerBranding) {
  const url = request.nextUrl.clone();
  url.pathname = "/login";
  return NextResponse.redirect(url);
}
```

**Defense-in-Depth bleibt erhalten:**
- Route-Handler validiert `partner_tenant_id` per UUID-RegEx (`route.ts:30`) → 400 bei ungueltigem Input
- RPC `rpc_get_branding_for_tenant` ist SECURITY DEFINER mit `SET search_path = public, auth` (DEC-109, R-091-4-Pattern)
- Storage-Read laeuft via Admin-Client gegen den unbenutzten Bucket `partner-branding-assets` (RLS bleibt fuer SELECT/INSERT/DELETE aktiv, Admin-Client umgeht das gewollt)

**Pattern-Konsistenz:** Der Guard liegt strukturell exakt analog zu `isApiCron` und `isApiUnsubscribe` — Middleware bleibt einheitlich lesbar.
