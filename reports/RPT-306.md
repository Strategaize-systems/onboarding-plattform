### Medium

**QA-F1 (Medium / V7.1-Polish)** — Vitest Test-Pollution bei standard-parallel-Mode
- **Symptom:** `npx vitest run <pen-test-files>` ohne `--no-file-parallelism` produziert 1-4 sporadische Failures pro Run (MT-5 Timing-Bound + MT-4 Case 18 Race-Counter + MT-6 Positiv-Probe). Mit Flag 3x stabil 25/25.
- **Root Cause:** Vitest Worker-Pool laeuft Test-Files parallel; alle 5 Pen-Test-Files teilen sich die Coolify-DB → INSERT-Pollution in `partner_organization` + `partner_client_mapping` + `error_log`. MT-5 Timing-Test misst Mean-Time auf der CPU, die durch parallele Worker verzerrt wird.
- **Workaround/Fix:** vitest.config.ts erweitern um `poolMatchGlob` mit `**/__tests__/pen-test.test.ts` + `service-key-timing.test.ts` + `dsgvo-no-plaintext.test.ts` in eigenem fork-Pool mit `singleFork: true`. ODER eigenes `npm run test:pen-test`-Skript fuer Slice-Schluss-/qa.
- **Vertieft RPT-305 F-3** (dort als "Volltree-Flaky" beschrieben, aber RPT-305-isolated-Run war per Vitest-Auto-Choice sequenziell — bei jeder CI/Volltree-Wiederholung wird das flaky).
- **NICHT SLC-134-Block** — Suite ist funktional korrekt (Anti-Abuse-Verifikation erbracht), Reliability-Polish ist V7.1.

**QA-F2 (Medium / Doku-Gap SLC-135)** — SLC-135 Live-Smoke-Plan ohne PostgREST-Schema-Reload
- **Symptom:** `slices/SLC-135-ttl-cleanup-cron-final-hardening-live-smoke.md` Z. 144-161 (13-Schritt-Live-Smoke-Plan) enthaelt keinen `NOTIFY pgrst, 'reload schema'`-Step nach Migration-Apply.
- **Auswirkung:** Bei /deploy V7 wird nach `Apply Migration 099+` der `/rest/v1/<tabelle>`-Endpoint ggf. fuer ~5min HTTP 404 statt 201/200 zurueckgeben (PostgREST cached Schema-Metadata) — Live-Smoke schlaegt mit nicht-zuordenbarer "Tabelle existiert nicht"-Fehlermeldung fehl, bis Auto-Reload greift.
- **Fix:** SLC-135 MT-6 (Live-Smoke-Plan-Doku) erweitern um Migration-Apply-Procedure mit explizitem `docker exec <db-container> psql -U postgres -d postgres -c "NOTIFY pgrst, 'reload schema';"` nach jeder Migration. Plus Reference auf existing Memory `reference_postgrest_schema_reload.md`.
