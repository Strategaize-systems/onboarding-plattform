## 5. Wiring Verification

Reine DML-Backfill-Migration auf einer einzigen Tabelle-Row → Wiring-Verification ist eingeschraenkt anwendbar:

- **Migration → DB**: `UPDATE template SET blocks=...::jsonb, sop_prompt=...::jsonb WHERE slug='mitarbeiter_wissenserhebung'` — Single-Row-UPDATE, idempotent, transactional.
- **DB → Capture-UI**: Capture-Session liest `template.blocks` → rendert Block-Titel + Question-Text. Mit korrigierten Umlauten zeigen UIs jetzt `würden`, `Rollenverständnis`, etc. statt `wuerden`, `Rollenverstaendnis`. Kein API-Routing-Change, kein neues Schema, kein neuer Endpoint.
- **DB → Worker**: Condensation-Worker liest `template.sop_prompt` als System-Prompt fuer Bedrock. Mit Umlauten-konsistentem Prompt erzeugen LLM-Outputs lesbareren deutschen Text — funktional identisch.
