## 4. Issues found

### ISSUE-102 (High) — entdeckt + gefixt + verifiziert in dieser Session
- **Befund:** Alle 3 V9.1-Coolify-Scheduled-Tasks (`inbound-email-imap-sync */5`,
  `email-bulk-pipeline-trigger` hourly, `bulk-email-retention-sweep` daily) feuern
  korrekt auf Cadence, scheitern aber bei JEDER Ausfuehrung mit
  `Job permanently failed after 1 attempts: sh: curl: not found`. Command nutzte
  `curl`; der Alpine-App-Container hat nur `wget`. Endpoints selbst funktionieren
  (manueller `docker exec wget` → `{success:true}`).
- **Effekt:** V9.1-Kern (continuous Inbound-Sync) seit ~08:18-Redeploy ~21h
  **dormant** — kein Auto-Pull aus IONOS. **Kein Datenverlust** (IONOS haelt Mail;
  `last_uid=1` resumed).
- **Root-Cause-Pin:** Aeltere April/Mai-Tasks (pending-signup-cleanup) succeeden
  (andere Command-Form); die 3 neuen V9.1-Tasks (curl) failen.
- **Fix:** Command-Rewrite in `coolify-db scheduled_tasks` von
  `curl -fsS -X POST <url> -H "x-cron-secret: $CRON_SECRET"` →
  `wget -qO- --post-data='' --header="x-cron-secret: $CRON_SECRET" <url>` fuer alle 3
  Tasks (alte Commands fuer Rollback gesichert; wget-over-HTTPS vorab im Container
  validiert).
- **Verifikation:** naechster `*/5`-Tick **06:10:02 UTC = success**
  `{"success":true,...,"lastUid":1}` (06:05/06:00 vorher = failed `curl: not found`);
  app `error_log` zeigt `cron:inbound-email-imap-sync run` 06:10:04 (Scheduler hat
  Endpoint erreicht). pipeline-trigger (hourly) + retention-sweep (daily) tragen den
  identischen validierten Fix, bestaetigen auf ihrem naechsten Tick.
