### V9.0 Scope (entschieden in /discovery)

**Persona**: GF im eigenen Tenant (Q-V9-1 entschieden). Nicht Mandant im Multiplikator-Pfad (Privacy-tief, V10+). Nicht Mitarbeiter (V9.2+). Nicht Customer-Service-Mitarbeiter (V10+ greenfield Konsum-Pfad).

**Daten-Quelle V9.0**: `.mbox`-Upload (Q-V9-2 entschieden — Modus C Hybrid). Format-Support: `.mbox` (Email-Client-Folder-Export aus Gmail-Takeout/Outlook-Export/Thunderbird/Apple Mail) + einzelne `.eml`. KEIN Forward-Bucket (V9.1+), KEIN IMAP (V10+), KEIN PST (V10+).

**Workflow V9.0**:

1. **Upload** — Drag-and-Drop von `.mbox` oder mehreren `.eml` (Reuse V2 Evidence-Multi-File-Upload-Pattern).
2. **KI-Pre-Filter-Pass (Haiku)** — Klassifikation pro Email in `content` / `short_reply` / `notification` / `newsletter` / `private` / `unclear`. Kosten ~0.0001 EUR/Email = ~0.10 EUR pro 1000 Emails.
3. **GF Filter-Review** — Liste mit "342 Emails: 87 content, 200 short_reply, ..." mit manueller Korrektur-Moeglichkeit.
4. **Thread-Aggregation** — `mailparser` + `In-Reply-To`/`References`-Headers gruppiert Emails zu Konversations-Threads (1 Thread = 1 Pattern-Extraktions-Einheit).
5. **PII-Redaction-Pass** — Reuse V5 SLC-076..078 Pipeline-Stufe (Klarnamen + Email-Adressen + Telefonnummern pseudonymisieren).
6. **Pattern-Extraktion-Pass (Sonnet)** — Pro Thread: Themen + wiederkehrende Antworten + Entscheidungen + offene Fragen. Nur auf `content` + optional `unclear` (90% Email-Volumen-Reduktion gegenueber Vollkorpus). Kosten ~5 EUR pro 1000 Emails (nach Pre-Filter).
7. **Pattern-Curation-UI** — GF reviewed Pattern-Liste: Akzeptieren / Ablehnen / Editieren / "in Handbuch-Section X einsortieren".
8. **Handbuch-Integration** — Akzeptierte Patterns fliessen als zusaetzliche `knowledge_unit`-Rows in V4.1 `handbook_snapshot` (Q-V9-3 entschieden).

**Konsum-Pfad V9.0**: Unternehmerhandbuch-Vervollstaendigung via V4.1 FEAT-028 Handbuch-Reader (Q-V9-3 entschieden). OP-internes Konsum-Ziel, KEIN IS-Knowledge-Push.

**Aufwand V9.0**: ~2-3 Wochen Code-Side + /qa + /final-check + /go-live + /deploy. Realistisch ~3-5 Slices: Upload+Parser, Pre-Filter, Thread-Aggregation+Redaction-Pipeline, Pattern-Extraktion+Curation-UI, Handbuch-Integration.
