## Problems found
- **IMP-051 (Lockfile-Drift):** Erster Coolify-Redeploy 2026-04-16 08:22 schlug fehl, weil `npm ci` die package.json-/lockfile-Konsistenz erzwingt. SLC-002a hatte neue devDeps eingefuehrt, ohne package-lock.json zu regenerieren. Fix: lokal `npm install --package-lock-only`, commit, push, Redeploy. Danach Build durchgelaufen.
- **IMP-052 (Next-Standalone-Trace-Luecke):** Erste Script-Variante importierte `@supabase/supabase-js`; das SDK bringt `@supabase/storage-js` -> `iceberg-js` mit, das Next.js-standalone nicht mit-tracet (weil Server-Code nur `@supabase/ssr` direkt importiert). Ein `COPY --from=builder /app/node_modules/@supabase/` im Dockerfile reicht NICHT, weil es die tiefer liegenden Transitive-Deps nicht zieht. Saubere Loesung: Script auf native fetch gegen REST + Admin-API umgebaut, Dockerfile-Extra-Copy zurueckgerollt. Kein SDK mehr im One-Shot-Script.
- **ISSUE-009 Blueprint-Profile-Flow Silent Failure:** Beim User-Smoketest entdeckt. Der `/profile`-Flow ruft nicht-existierende `owner_profiles`-Tabelle auf (per MIG-003 entfernt). `handleSave` prueft nur `res.ok === true` und zeigt bei Fehlern gar nichts. User blockiert. Bewusst nicht in SLC-002b gefixt — Rule 4 (architektonische Entscheidung: Blueprint-Feature entfernen vs. nachliefern). User hat Option A (entfernen) gewaehlt. In SLC-002d geplant.
