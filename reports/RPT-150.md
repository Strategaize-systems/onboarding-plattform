### Beobachtung Check 3: HTTP 200 statt 307

RPT-149 hatte fuer `/login` HTTP 307 erwartet. `/login` ist aber die tatsaechliche Login-Page und liefert HTTP 200 mit `text/html` Content-Type. Ein 307-Redirect erscheint, wenn ein protected Route (z.B. `/dashboard`) zu `/login` umleitet — nicht aber, wenn man `/login` direkt aufruft. Spec war hier ungenau, das tatsaechliche Verhalten ist korrekt. PASS.

### Beobachtung Worker-Log-Tail: ESM-Warning

Im Worker-Container erscheint: `(node:1) [MODULE_TYPELESS_PACKAGE_JSON] Warning: Module type of file:///app/worker.js is not specified and it doesn't parse as CommonJS. Reparsing as ES module because module syntax was detected.`

Das ist eine reine Performance-Hint-Warning — Worker laeuft funktional einwandfrei. Fix-Option: `"type": "module"` in `/app/package.json` ergaenzen. Nicht Release-blockierend; als IMP fuer V4.4-Maintenance-Sweep festgehalten.
