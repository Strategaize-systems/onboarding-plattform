### Bug entdeckt + gefixt (Schema-Drift im Code, nicht in DB)

**Symptom:** Detail-Page rendert N=0 obwohl DB 4 Schritte + 4 Mappings hatte.

**Diagnose via wget aus app-container:**
- `walkthrough_step?...&select=id,...,reviewer_corrected` → **HTTP 400 Bad Request**
- `walkthrough_step?...&select=id,...` (ohne reviewer_corrected) → 4 Schritte korrekt
- `\d walkthrough_step` zeigte: KEIN `reviewer_corrected`-Feld (nur in walkthrough_review_mapping)

**Root cause:** Mein /frontend SLC-079-Code hatte `reviewer_corrected` faelschlich an walkthrough_step erwartet (im Server-Side `.select()`-String + im StepCardData-Interface). Migration 085 + Architecture haben das Feld korrekt nur in walkthrough_review_mapping. Mein Code destructured `{ data: stepRows }` ohne `error`-Check → supabase-js verschluckt den 400-Fehler silent → leeres Array → N=0-Render-Branch.

**Fix:** Commit `3a231e9` — `reviewer_corrected` aus dem step-`.select()`-String + StepCardData-Interface entfernt. `mapping.reviewer_corrected` (in walkthrough_review_mapping) wird wie bisher korrekt fuer das "Berater-korrigiert"-Badge benutzt.

**Coolify-Workflow-Issue:** User musste 4× redeployen, bis der neueste Commit gebaut wurde — Coolify "force deploy" hat 2× nur den vorherigen Image-Hash neu gestartet ohne git-pull. Verifikation via `docker images` zeigte den richtigen Tag erst beim 4. Versuch.
