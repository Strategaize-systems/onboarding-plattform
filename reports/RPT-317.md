### Low

- **Low-1: `npm run build` Page-Data-Collection schlaegt fehl mit `supabaseUrl is required`.** Compile + TypeScript laufen sauber durch (`✓ Compiled successfully in 11.6s`). Page-Data-Collection scheitert auf `/api/public/partner/[slug]` weil im Worktree kein `.env.local` existiert. Die Route stammt aus SLC-131/134 (commit `c99c8a1` / `610d20f`), nicht aus SLC-136. Diff `main...HEAD -- src/app/api/public/partner/` ist leer. Verifikation: Auf der Production-Container-Instanz mit gesetzten Supabase-ENVs ist diese Page-Data-Collection nachweislich gruen (V7-Release REL-020). Empfehlung: keine Action — Worktree-spezifisch.

- **Low-2: Policy-Redundanz bei SELECT global+template.** `text_override_partner_read_global_template` (Policy 3b) ist FOR SELECT mit `USING (scope IN ('global','template'))` ohne Rolle-Filter. `text_override_tenant_read` (Policy 3d) ist FOR SELECT mit `USING (scope IN ('global','template') OR ...)`. 3b ist Subset von 3d. Permissive-OR-Merge = funktional korrekt, semantisch redundant. Kein Bug. Cleanup-Option fuer V7.2-Slice falls Policy-Set neu sortiert wird. Keine Action notwendig.

- **Low-3: AC-SLC-136-3 Live-Browser-Smoke deferred.** Der Code-Path ist deterministisch (Server-Component + RLS-DB-Filter) und durch 17/17 RLS-Pen-Tests live verifiziert. Live-Browser-Smoke wird per `feedback_slice_merge_at_end.md` + RPT-316 Open Point #3 nach Master-Merge + Coolify-Redeploy nachgeholt. Risiko = niedrig (kein Client-State, kein Browser-spezifisches Verhalten). Nachfass-Schritt: nach Master-Merge auf `app.onboarding-plattform.de/admin/text-overrides` mit strategaize_admin- + partner_admin-Session smoken (Empty-State, Filter, History-Link, Reset-Button).
