## Problems Found

### Spec-Korrektur 1: computeSui-Formel hatte Scale-Bug

Slice-Spec Z. 259 (Original): `computeSui(moduleScores): number — Gewichtetes Mittel (m1*10 + ... + m8*10 + m9*20) / 100, Returns 0-100`.

Realitaet: bei Module-Scores 0-10 und Gewichtungen 10/.../20 (Summe 100) liefert `sum/100` nur 0-10 — nicht 0-100. Beispiel: alle Module = 10 -> Summe = 10*100 = 1000 -> /100 = 10. Test "Alles Stufe 5 -> SUI=100" wuerde fehlschlagen.

Korrekte Formel: `sum / 10` (algebraisch `weighted_mean * 10`). Implementierungs-Form vereinfacht zu `sum(m1..m8) + m9*2`. AC-Asserts (50/100/80/20) PASS.

**Anwendung in MT-6**: Server-Action `finalizeMandantenReport` ruft `computeSui(moduleScores)` direkt auf, keine externe Formel-Referenz noetig.

### Spec-Korrektur 2: selectThreeHebel-Signatur braucht 3. Arg

Slice-Spec Z. 264 (Original): `selectThreeHebel(moduleScores, stufenLookup): Array<{ modul_id, modul_name, score, stufe, empfehlung }>`.

Realitaet: Output-Feld `modul_name` ist im `stufen_lookup` nicht enthalten — `stufen_lookup` hat nur `{was_es_bedeutet, unsere_empfehlung}` Strings pro Stufe. Modul-Namen leben in `template.blocks[i].name`.

Loesung: 3. Arg `modulNames: Record<ModulKey, string>` zwingend. In MT-6 wird Caller extrahieren via `Object.fromEntries(template.blocks.filter(b => /^M[1-9]$/.test(b.modul_id)).map(b => [b.modul_id.toLowerCase(), b.name]))`.

Alternative waere gewesen, `template` als 3. Arg zu uebergeben (gibt 2-Arg-Caller die Wahl, das selber zu extrahieren), aber `modulNames` ist enger im Vertrag und einfacher zu testen.

### Spec-Description Mismatch bei mapModuleScoreToStufe

Slice-Spec Z. 261: `Bereichs-Mitten als Schwellen: 0-1 -> 1, 1.01-3 -> 2, 3.01-6 -> 3, 6.01-9 -> 4, 9.01-10 -> 5`.

Spec-Description sagt `0-1 -> 1` (inklusiv-1) — das wuerde `score=1 -> Stufe 1` heissen.

AC-Test Z. 271: `Bereichs-Mitten 1/4/7 -> 2/3/4` — sagt `score=1 -> Stufe 2`.

AC ist Ground Truth (in Vitest verifizierbar). Implementierte Grenzen: `<1 -> 1, [1,3] -> 2, (3,6] -> 3, (6,9] -> 4, >9 -> 5`. Midpoint-Tie-Up-Semantik (1 ist Midpoint zwischen Stufen-Score 0 und 2 -> rundet hoch).
