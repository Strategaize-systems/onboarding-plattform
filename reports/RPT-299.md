- **P-1: Schema-Drift `logo_url`/`accent_color`.** Slice + ARCHITECTURE.md
  V7-Sketch sprachen von `SELECT display_name, logo_url, accent_color FROM
  partner_organization`. Tatsaechliches Schema: nur `partner_organization.
  display_name` existiert; `logo_url` + `primary_color` (Slice nennt es
  `accent_color`) liegen in `partner_branding_config`. **Auto-Fix in MT-6**:
  Endpoint macht 2-stufigen Lookup (partner_organization → tenant_id →
  partner_branding_config LEFT JOIN), Default-Color `#4454b8` (Style-Guide-
  V2) wenn keine Branding-Row, `pbc.display_name` ueberschreibt
  `po.display_name` wenn gesetzt. Architecture-Doc-Update als optionales
  Follow-Up in /docs.
- **P-2: 135 Vitest-Regressions nach Migration 097 NOT NULL.** Pre-existing
  V6-DB-Tests in 7 Files (mandanten + branding + accept-invitation +
  partners + lead-push + partner-stammdaten + v6-partner-rls) inserten
  raw-pg `partner_organization` ohne slug-Wert. **Auto-Fix in MT-7**:
  Migration 097a setzt `DEFAULT ('p-' || gen_random_uuid())` auf der
  Spalte, sodass Legacy-Inserts automatisch einen unique Slug bekommen.
  Production-Code setzt slug immer explizit, DEFAULT kommt dort nicht
  zum Zug. Trade-off: ad-hoc-SQL-Inserts ohne explizite Slug-Werte bekommen
  einen `p-<uuid>`-Slug — unscharf aber kein Schaden.

  maybeSingle()` umgestellt wurde. Auf MAIN gleiche 2 Failures (geprueft).
