#### F-1: Auto-Fix — defaultScopeForKey returned scope='template'|'partner' (RESOLVED per Deviation Rule 1)

**Issue**: `src/components/text-override/editable-text-logic.ts:defaultScopeForKey`
mapped `template.*`-Keys auf `scope='template'` und `partner.*`-Keys auf
`scope='partner'`. Beide setzen `scope_id=null` im UI-Code, was von
`actions.ts:validateScope` mit `scope_id_required_for_template_or_partner`
abgelehnt wird → Edits auf template.*-Keys haetten in Production gar nicht
funktioniert.

**Spec-Drift**: FEAT-056 Props-Section sagt explizit `scope?: ... (Default
'global')`. Helper-Default war Drift gegen die Feature-Spec.

**Fix (Auto per Deviation Rule 1 — Bug-Fix)**: `defaultScopeForKey` returns
immer `'global'`. Caller kann scope-Prop explizit auf `'partner'` setzen
(scope_id automatisch via `ctx.partnerOrgId`) oder auf `'template'` (dann
muss `scopeId`-Prop manuell uebergeben werden, z.B. von einer Page mit
Template-Kontext). Strategaize-Admin kann via `/admin/text-overrides`
spaeter Scope re-zoomen.

**Tests aktualisiert**: 3 Vitest-Cases zu `defaultScopeForKey` neu (alle 3 PASS).
EditableText.tsx-Kommentar bereinigt.

**Commit**: `86a5daa fix(SLC-137/qa-auto-fix): defaultScopeForKey always 'global' (Deviation Rule 1)`

#### F-2: AC-SLC-137-3 / AC-11 Performance Live-Verify deferred auf Post-Master-Merge

**Issue**: AC-11 "Server-Render mit 50 EditableText < 100ms" ist Code-Side
PASS-by-Design (Single-Query-Load via `loadOverridesWithCache` mit 60s-
in-memory-Cache + O(1) `selectEffectiveText` per Call → 48 calls = sub-ms
CPU). Live-Verify mit Browser-DevTools-Profiler / React-Profiler benoetigt
Production-Build auf Coolify.

**Decision**: Deferred auf Post-Master-Merge Live-Smoke. Acceptable weil
Single-Query-Load + Map-Cache architekturell die Performance-AC erfuellt;
Drift waere nur via Bug im Resolver moeglich (existing SLC-136 Foundation
mit 30/30 Vitest gegen Coolify-DB live verified).
