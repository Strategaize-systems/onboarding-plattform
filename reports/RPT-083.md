## Problems found

1. **Layout-Konflikt zwischen /employee/layout.tsx und QuestionnaireWorkspace fullscreen.**
   - **Symptom:** QuestionnaireWorkspace nutzt `flex h-screen` (100vh, ignoriert Layout-Hierarchie). Mit dem urspruenglichen `<main className="mx-auto max-w-5xl px-6 py-10">`-Wrapper aus /employee/layout.tsx haette der Workspace einen begrenzten Container statt Vollbild gehabt.
   - **Fix:** Layout vereinfacht (nur Header + flex-1 main ohne Container). Listen-Pages (/employee, /employee/capture/[sessionId]) wrappen sich selbst in `mx-auto max-w-5xl px-6 py-10`. Block-Detail-Page rendert fullscreen Workspace ohne Wrapper.
   - **Folge-Implikation fuer QA:** Beim Block-Detail-Aufruf ist der Mitarbeiter-Header (logo + Logout) hinter dem h-screen-Workspace verdeckt. Logout-Pfad fuer Mitarbeiter im Capture-Mode geht ueber "Zurueck zur Uebersicht" ‚Üí /employee ‚Üí Logout. **Dies ist eine bewusste UX-Konzession (Slice MT-1 sagt explizit "restringiertes Navigations-Menu").**

2. **validation_layer-CHECK reviewer_role nur 'strategaize_admin' oder 'tenant_admin'.**
   - **Befund:** `reviewer_role` in validation_layer (Migration 021) erlaubt employee-Inserts NICHT. RLS-Matrix-Test fuer employee SELECT validation_layer funktioniert trotzdem, weil seedV4Fixtures als postgres-Superuser inserted (BYPASSRLS).
   - **Implication:** Wenn jemals ein employee selbst INSERT in validation_layer machen sollte, wuerde der CHECK fehlschlagen. Aktuell entstehen validation_layer-Rows nur via `rpc_update_knowledge_unit_with_audit` (SECURITY DEFINER), und der RPC wird nur fuer tenant_admin/strategaize_admin aufgerufen. Konsistent mit aktueller Architektur, aber zukuenftige employee-Edit-Flows muessten das beachten.

3. **Tests benoetigen TEST_DATABASE_URL.** Ohne TEST_DATABASE_URL wirft `withTestDb` einen Error ‚Üí 43 RLS-Tests faellen ins Setup. `npm run test` muss in der QA-Phase mit `TEST_DATABASE_URL=postgres://...@onboarding-coolify-db/...` aufgerufen werden. Lokales WSL/dev kann `npm run test` ohne TEST_DATABASE_URL aufrufen ‚Äî die Tests skippen oder error-out, das ist erwartet.
