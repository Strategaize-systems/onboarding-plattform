## Fix-Trace

**Vor dem Fix (RPT-261 Wiring-Gap):**

```
src/app/layout.tsx           → createClient()      # Instanz A
src/app/dashboard/page.tsx   → createClient()      # Instanz B (≠ A)
                                ↓
resolveBrandingForTenant(supabase, tenantId)        # cache() Args-Memo
   → Object.is(A, B) === false → Cache-MISS         # 2x RPC pro Request
```

**Nach dem Fix:**

```
src/app/layout.tsx           → createClient()      # cache()-Memo, Instanz A
src/app/dashboard/page.tsx   → createClient()      # cache()-Hit, Instanz A
                                ↓
resolveBrandingForTenant(supabase, tenantId)        # cache() Args-Memo
   → Object.is(A, A) === true  → Cache-HIT          # 1x RPC pro Request
```

Damit ist der von RPT-261 dokumentierte Wiring-Gap geschlossen: dieselbe `SupabaseClient`-Instanz fliesst aus Layout und Page in den Resolver, die Object.is-Args-Memoization in `resolveBrandingForTenant` greift jetzt korrekt.
