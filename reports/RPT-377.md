## Problems Found

**F-1 (Low — dokumentiert, NICHT blockierend)**: `npm run build` Phase-3 (Collecting page data) faellt mit `Error: supabaseUrl is required` aus `/api/public/partner/[slug]` route. Ursache: `.env*.local` ist im Repo `.gitignore`d (Pattern `.env*.local`), und der Parent-Tree hat selbst nur `.env.local.example` (keine echten ENV-Files). Page-Data-Collection ist Next.js-Build-Time-Pflicht, braucht `NEXT_PUBLIC_SUPABASE_URL` oder `SUPABASE_URL` zur Modul-Evaluation. **Bedeutung fuer MT-0**: Code-Level-Baseline (TSC + Compile) ist PASS — keine Regression in Baseline-Code. Die Build-Phase-3 ist Coolify-Time-Concern (Coolify setzt ENV-Vars beim Deploy). Lokales Full-Build ist nicht reproduzierbar ohne `.env.local`-Setup, identisch fuer Parent + Worktree. **Mitigation**: TSC `--noEmit` als Code-Baseline-Check etabliert (EXIT=0). Wenn /qa Local-Full-Build benoetigt → User muss `.env.local` von Coolify-Coolify-Resource-ENV exportieren ODER Test-Stubs setzen.

**F-2 (Info — out of MT-0-Scope)**: `.env.deploy.example` (7.3 KB) + `.env.local.example` (2.8 KB) existieren im Parent + jetzt Worktree als Template-Files. Sind in Git getrackt (kein gitignore-Match). Coolify-Deploy nutzt vermutlich `.env.deploy` (untracked aber existiert auf Server), Dev-Local-Build wuerde `.env.local` aus `.env.local.example` ableiten. Nicht MT-0-Scope, aber nuetzlich fuer kuenftige /qa-Local-Smoke-Sessions zu wissen.

**F-3 (Info — Lessons Learned)**: Im Initial-Slice-Spec Verification "`npm run build` PASS auf Baseline" wurde implizit angenommen dass die Local-Build-Pipeline ohne ENV-Setup-Overhead funktioniert. Empfehlung als IMP fuer Dev-System: Slice-Spec-MT-0-Vorlagen sollten `npm run build` durch `npx tsc --noEmit` als Default-Baseline-Check ersetzen (TSC reicht aus, Next-build-PageData ist Coolify-Time, lokal nur mit ENV reproduzierbar). Siehe Recommended next step.
