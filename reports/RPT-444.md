# SLC-V9.1-D /frontend — Setup-UI Frontend Layer

## Outcome
**Partially complete (frontend layer done; slice /qa + MT-7 docs + closure pending).** Built the full Conversational-First Setup-UI for the Forward-Bucket-Email feature on top of the already-committed backend layer (RPT-443): a server-component page, a client orchestrator, the 5 named components, and a thin auth-gated `suggestSetup` server-action wrapper for the Bedrock assistant. SLC-V9.1-D stays `in_progress`. No master-merge, no `released` (per guardrails / IMP-950 / module-lifecycle-discipline).

## What was done
- **page.tsx (server)** — tenant_admin auth-gate (pattern from `bulk-email-import/page.tsx`), RLS-scoped read of the tenant's `email_inbound_endpoint` (+ `email_forward_allowlist`), builds the `bulk-<slug>@<INBOUND_CATCHALL_DOMAIN>` address and hands a typed `ExistingEndpoint | null` to the client orchestrator.
- **ForwardSetupWizard (client orchestrator)** — two phases: (A) no endpoint → assistant + create-form; (B) endpoint exists → token/address, mail-client instructions, allowlist manager, test-send, DSGVO activation. Wires all 5 server-actions (`createInboundEndpoint`, `regenerateSetupToken`, `updateAllowlist`, `sendTestEmail`, `confirmDsgvoDisclaimer`) + `suggestSetup`. Assistant suggestions pre-fill local-part and best-effort auto-add suggested allowlist patterns after create.
- **ConversationalSetupAssistant (MT-3b)** — Conversational-First (BLOCKING). NL textarea **+ voice input** (browser `MediaRecorder` → `POST /api/tenant/transcribe`, returns `{ transcript }`) — voice pattern ported 1:1 from the in-repo `questionnaire-form.tsx` (no KI-assistant existed in OP; chose in-repo voice reuse over BS/IS port for consistency). Calls `suggestSetup`, renders local-part + allowlist-badges + reasoning, `onApply` → wizard prefill.
- **SetupTokenDisplay (MT-1)** — address + once-only setup-token with copy buttons and "shown only now" warning.
- **MailClientInstructions (MT-1)** — tabbed step-by-step forwarding setup for Gmail / Outlook-M365 / IONOS Webmail (shadcn Tabs).
- **DsgvoDisclaimerModal (MT-4)** — Dialog + checkbox gate; confirms with `DSGVO_CONSENT_TEXT_VERSION` from shared `dsgvo-consent.ts`; success → endpoint `active`.
- **TestSendButton (MT-4)** — `sendTestEmail` + poll-result feedback (received / not-yet / error), `onReceived` hook.
- **suggestSetup server-action** — added to `forward-setup/actions.ts`: auth-gated wrapper around `summarizeSetupIntent` (the backend helper has no `"use server"`, so a client-callable async wrapper was required). error_log audit via `captureInfo`/`captureException`, `SetupSuggestionError` → friendly message.
