## Problems found

1. **Stack-Limitation: kein happy-dom / @testing-library/react.** Vitest laeuft in node-env. Component-Rendering-Tests waeren ohne neue Dev-Deps nicht moeglich. **Mitigation**: Pure-Logic-Helper extrahiert in `editable-text-logic.ts`, 23 Vitest-Cases auf Helper. Visual-Verification auf /qa-Live-Smoke (Pencil-Icon-Visibility pro Rolle, Inline-/Modal-Open, Save-Round-Trip) verschoben. Slice-Spec MT-1 verlangt "8+ Cases" — mit 23 weit ueberschritten in Pure-Logic-Coverage. **Empfehlung V7.2**: @testing-library/react + happy-dom als Dev-Deps adden wenn Component-Tests Pflicht werden.

2. **Volltree-Vitest scheitert ohne TEST_DATABASE_URL.** 35 von 118 Test-Files fail beim `npx vitest run` mit "TEST_DATABASE_URL is not set". Sind pre-existing DB-Integration-Tests die gegen Coolify-Postgres laufen sollen (siehe `[[reference_coolify_test_setup]]`). **Nicht SLC-137-bedingt**. SLC-137-relevant: 5 Files / 61 Tests alle PASS (siehe Quality-Gate-Tabelle).

3. **template-scope scope_id-Resolution open.** EditableText speichert `scope='template'`-Keys mit `scope_id=null` (siehe Code-Kommentar in EditableText.tsx). Slice-Spec MT-2 erwaehnt "V7.1 V1: nur partner_diagnostic-Template". Save mit scope=template + scope_id=null wuerde von actions.ts.validateScope rejected ("scope_id_required_for_template_or_partner"). **Impact**: Edits auf template-Keys schlagen aktuell mit "scope_id_required..." fehl. **Loesung kommt in /qa**: defaultScopeForKey-Helper liefert "template" fuer template.* keys, aber Save braucht scope_id=template.id. Wir muessten entweder die scope_id auf eine konkrete UUID setzen (was wir nicht zur Hand haben im Client) oder im Save-Path scope auf "global" mappen. **V7.1-Workaround vorgeschlagen in /qa**: Default-Save mit scope="global" fuer ALLE template.*-Keys, scope="partner" nur fuer Partner-Branding. Pencil-Icon zeigt + Save funktioniert. Strategaize-Admin kann in /admin/text-overrides spaeter manuell auf scope=template re-zoomen. → Doku-Update in /qa.

4. **AC-SLC-137-3 Performance-Live-Verify deferred.** "Server-Render mit 50 EditableText < 100ms" ist code-side via Single-Query-Load + 60s-Cache implementiert, aber nicht messbar ohne Production-Build + Hetzner-Smoke. Deferred auf Post-Master-Merge-Live-Smoke.

5. **Audit-Skript Heuristik nicht 100% praezise.** Mehrere Filter-Layers (Tailwind-Classes, SELECT-Spalten-Listen, EditableText-Args, Browser-Tab-Titel) reduzieren False-Positives von 501 (Baseline-Run) auf 138. Verbleibende 138 enthalten ~30 echte Hardcodes in Out-of-Scope-Files (IchWillMehrModal/StatusCard) + ~100 in Email-Templates die als Subject/Body-Pairs zaehlen sollten + ~10 false positives (Error-Messages in Server-Actions etc.). **Acceptable**: AC-1 fordert Count ≥ 50 — Audit zeigt 66 → AC erfuellt.
