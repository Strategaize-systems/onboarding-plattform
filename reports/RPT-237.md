## Problems Found

### F1 — Strategaize-Default-Color Drift zwischen Migration-Source 091 und deployed RPC

Erst-Run produzierte 12 FAILs in den Default-erwartungs-Cases mit Diff `Expected '#2563eb' / Received '#4454b8'`. Live-RPC-Body via `pg_get_functiondef` zeigt:
```
'primary_color', '#4454b8',  -- SLC-104 MT-6 (091a): aligned to Style Guide V2 (was '#2563eb' in 091)
```
Migration 091 (Source) hat noch `#2563eb`; deployed Function nutzt `#4454b8` (Style-Guide-V2-Indigo, eingefuehrt durch `091a_align_strategaize_default_brand_color_styleguide_v2.sql` in MT-6 via `ALTER FUNCTION`/`CREATE OR REPLACE`).

**Root Cause:** Migration-Source-Files bleiben historisch ueber 091a-Patch, was korrekt ist (Migrations sind append-only). Aber Tests, die "Erwartung = Source-Datei" annehmen, sehen die nachtraegliche Anpassung nicht.

**Resolution:** `STRATEGAIZE_DEFAULT.primary_color` im Test auf `#4454b8` korrigiert + Kommentar dass dies das 091a-Alignment ist (Style-Guide-V2-Indigo-500), nicht 091-Original-Tailwind-Blue.

**Dev-System-Process-Finding:** kandidat fuer IMP — siehe Section "Dev-System Skill-Improvements".

### F2 — Orphan partner_client per CHECK-Constraint verhindert

Erst-Run produzierte 1 ERROR in Edge Case 3 (Orphan-partner_client): `new row for relation "tenants" violates check constraint "tenants_parent_partner_consistency"`. Constraint-Definition:
```
CHECK (((tenant_kind = 'partner_client' AND parent_partner_tenant_id IS NOT NULL)
        OR (tenant_kind <> 'partner_client' AND parent_partner_tenant_id IS NULL)))
```
Die RPC enthaelt einen defensiven ELSE-Branch fuer NULL-parent — dieser ist per Schema-Design unreachable. Beruhigend, aber Test muss das wissen.

**Resolution:** Edge Case 3 umformuliert: testet jetzt, dass der INSERT durch den CHECK-Constraint abgewiesen wird (SAVEPOINT-Pattern). Die unreachable RPC-Branch ist als bewusste Defense-in-Depth dokumentiert.
