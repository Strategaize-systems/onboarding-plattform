## Issues Found

### ISSUE-072 (BLOCKER, NEU)
**Externer HTTPS-Routing zur App-Resource haengt — Multi-Network-Falle.**

**Diagnose:**
- App-Container ist auf 2 Networks: `bwkg80w04wgccos48gcws8cs` (IP=10.0.3.18) + `bwkg80w04wgccos48gcws8cs_strategaize-net` (IP=10.0.4.17)
- Coolify-Proxy ist nur auf `bwkg80w04wgccos48gcws8cs` + `coolify`
- App-Labels: keine `traefik.docker.network=bwkg80w04wgccos48gcws8cs` + keine `traefik.http.services.X.loadbalancer.server.port=3000`
- → Traefik waehlt das falsche Backend-Interface (`10.0.4.17` strategaize-net, das er nicht erreichen kann) → Backend-Connect haengt → 504 nach Timeout

**Empirische Verifikation:**
```
docker exec coolify-proxy wget http://10.0.3.18:3000/login   → HTTP 200 OK
docker exec coolify-proxy wget http://10.0.4.17:3000/login   → Hang
```

**Onset-Window:**
RPT-256 (2026-05-14 ~14:58 UTC) verifizierte 88ms TTFB ✓. RPT-252 (2026-05-14 Cross-System-Smoke) verifizierte E2E ✓. Heute 2026-05-15 ~06:30 UTC: Hang. Coolify-Sentinel restart 2026-05-14 ~23:30 UTC ist potenzieller Trigger fuer Network-Reorder.

**Pattern bekannt aus:**
- memory `feedback_coolify_multi_network_traefik.md` (2026-04-19 BS V4.1 Lehre)
- rule `jitsi-jibri-deployment.md` Punkt 3 (Multi-Network → `traefik.docker.network` Pflicht)
